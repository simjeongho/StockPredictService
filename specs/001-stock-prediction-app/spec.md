## User Scenarios & Testing *(mandatory)*

### User Story 1 - 종목 검색 및 대시보드 조회 (Priority: P1)

개인 투자자가 관심 있는 종목을 검색하면 해당 종목의 현재가, 등락률, 거래량 등 핵심 정보와 함께 최대 1년치 주가 차트를 즉시 확인할 수 있다. 검색은 종목명(예: 삼성전자, Apple) 또는 티커 심볼(예: AAPL, 005930)로 모두 가능하다.

**Why this priority**: 모든 다른 기능(AI 분석, 관심 종목, 챗봇)의 진입점이며, 이 기능 없이는 서비스 자체가 성립하지 않는다. 단독으로도 차트 조회 도구로서 완결된 가치를 제공한다.

**Independent Test**: 사용자가 티커(예: AAPL)를 검색하면 현재가·등락률·거래량 및 최근 1개월 차트가 1분 이내에 표시된다. 로그인 없이도 이 흐름이 완결되면 독립적으로 검증 가능하다.

**Acceptance Scenarios**:

1. **Given** 사용자가 검색창에 "AAPL"을 입력하고 검색을 실행했을 때, **When** 시스템이 결과를 반환하면, **Then** Apple Inc.의 현재가, 전일 대비 등락률(%), 거래량, 시가총액이 화면에 표시된다.
2. **Given** 사용자가 "삼성전자"를 입력하고 검색했을 때, **When** 결과가 반환되면, **Then** KOSPI 종목인 삼성전자(005930)의 기본 정보가 표시된다.
3. **Given** 종목 상세 화면에서, **When** 사용자가 "1개월" 탭을 선택하면, **Then** 최근 1개월 주가 차트가 표시된다. 동일하게 3개월/6개월/1년 탭도 동작한다.
4. **Given** 사용자가 존재하지 않는 티커(예: "XXXXXX")를 검색했을 때, **When** 검색 결과가 없으면, **Then** "해당 종목을 찾을 수 없습니다" 메시지가 표시된다.
5. **Given** 시장 데이터가 일시적으로 불가능한 상태일 때, **When** 사용자가 검색하면, **Then** 마지막으로 수신된 데이터와 타임스탬프가 함께 표시되고, 실시간 데이터가 아님을 명시한다.

### User Story 2 - AI 기술적 분석 리포트 (Priority: P2)

사용자가 특정 종목의 AI 분석을 요청하면, 이동평균선·RSI·MACD·볼린저 밴드 등 실제 기술 지표 데이터를 기반으로 AI가 분석 리포트를 생성한다. 단기(1주) 및 중기(1개월) 전망이 텍스트로 제공되며, 모든 분석에는 면책 고지가 포함된다.

**Why this priority**: 서비스의 핵심 차별화 요소이며, 사용자가 AI 보조 분석을 받으러 방문하는 주된 이유다. 대시보드(P1) 이후 자연스러운 다음 단계다.

**Independent Test**: 종목 상세 화면에서 "AI 분석 요청" 버튼을 누르면 1분 이내에 기술 지표 기반 분석 텍스트가 표시된다. 해당 텍스트에는 단기·중기 전망과 면책 고지가 포함되어야 한다.

**Acceptance Scenarios**:

1. **Given** 사용자가 AAPL 종목 상세 화면에서 "AI 분석" 버튼을 클릭했을 때, **When** 분석이 완료되면, **Then** 이동평균선·RSI·MACD·볼린저 밴드 지표 데이터를 근거로 한 분석 텍스트가 1분 이내에 표시된다.
2. **Given** AI 분석 리포트가 생성되었을 때, **When** 사용자가 결과를 확인하면, **Then** 단기(1주) 전망과 중기(3개월) 전망이 별도 섹션으로 구분되어 표시된다.
3. **Given** AI 분석 결과 화면에서, **When** 어느 위치에서든, **Then** "본 분석은 투자 참고용이며, 투자 판단의 책임은 사용자에게 있습니다"라는 면책 고지가 항상 표시된다.
4. **Given** 분석에 사용된 기술 지표 데이터가 오래되었거나 불완전할 때, **When** AI 분석이 요청되면, **Then** 시스템은 분석 생성을 거부하고 데이터 부족 이유를 사용자에게 안내한다.

### User Story 3 - 관심 종목 관리 (Priority: P3)

로그인한 사용자가 관심 종목을 등록하고 목록으로 관리할 수 있다. 관심 종목 요약 화면에서는 등록된 모든 종목의 현재가·등락률 변동 사항을 한눈에 볼 수 있다.

**Why this priority**: 챗봇·비교 점수 기능(P4, P5)의 전제 조건이며, 개인화 서비스의 기반이다. P1·P2 이후 사용자가 서비스에 지속적으로 방문하게 만드는 retention 기능이다.

**Independent Test**: 사용자가 종목 상세 화면에서 "관심 종목 추가" 버튼을 클릭하면 관심 종목 목록에 저장되고, 관심 종목 요약 화면에서 해당 종목의 현재 정보가 표시된다.

**Acceptance Scenarios**:

1. **Given** 로그인한 사용자가 TSLA 상세 화면에서, **When** "관심 종목 추가"를 클릭하면, **Then** TSLA가 관심 종목 목록에 저장되고, 버튼이 "관심 종목 해제"로 변경된다.
2. **Given** 사용자가 관심 종목 요약 화면으로 이동했을 때, **When** 화면이 로드되면, **Then** 등록된 모든 관심 종목의 현재가·등락률·거래량이 리스트 형태로 표시된다.
3. **Given** 사용자가 관심 종목 목록에서 특정 종목의 "삭제" 버튼을 클릭했을 때, **When** 확인이 완료되면, **Then** 해당 종목이 관심 종목 목록에서 제거된다.
4. **Given** 비로그인 사용자가 "관심 종목 추가"를 클릭했을 때, **When** 버튼이 클릭되면, **Then** 로그인 또는 회원가입 화면으로 안내된다.

### User Story 4 - AI 챗봇 질의응답 (Priority: P4)

사용자가 특정 종목에 대해 자유로운 자연어 질문을 입력하면, AI가 해당 종목 관련 정보를 기반으로 자연어로 응답한다. 예: "삼성전자 최근 실적은 어때?", "이 종목의 지지선은 어디야?"

**Why this priority**: 대화형 분석은 고급 기능이며, P1–P3 흐름이 동작한 이후에 추가 가치를 제공한다. 독립적으로도 유용하지만 다른 기능들 위에서 시너지가 극대화된다.

**Independent Test**: 종목 상세 화면(또는 챗봇 전용 화면)에서 "이 종목의 RSI는?" 같은 질문을 입력하면 AI가 해당 종목 데이터를 참조하여 자연어로 답변한다.

**Acceptance Scenarios**:

1. **Given** 사용자가 AAPL 관련 화면에서 챗봇에 "최근 1개월 주가 흐름을 요약해줘"라고 입력했을 때, **When** AI가 응답하면, **Then** AAPL의 최근 주가 데이터를 기반으로 한 요약 텍스트가 반환된다.
2. **Given** 사용자가 지원되지 않는 질문(예: "오늘 날씨 어때?")을 입력했을 때, **When** AI가 응답하면, **Then** 주식 분석 관련 질문만 지원한다는 안내 메시지가 반환된다.
3. **Given** 챗봇 응답이 생성되는 동안, **When** 사용자가 화면을 보고 있으면, **Then** 로딩 상태가 명확히 표시된다(예: 타이핑 인디케이터 또는 로딩 스피너).
4. **Given** AI 챗봇 응답에서, **When** 어느 응답에서든 투자 판단에 영향을 줄 수 있는 내용이 포함되면, **Then** 면책 고지가 응답 하단에 포함된다.

---

### User Story 5 - 관심 종목 예측 점수 비교 (Priority: P5)

관심 종목으로 등록된 주식들에 대해 단기(1주)·중기(3개월)·장기(1년) 관점의 AI 예측 점수를 100점 만점으로 산출하여, 사용자가 종목 간 비교 화면에서 한눈에 확인할 수 있다.

**Why this priority**: 관심 종목(P3)이 전제 조건이며, 가장 복잡한 기능이다. P1–P4 기반이 탄탄해진 이후 추가할 때 가장 큰 가치를 발휘한다.

**Independent Test**: 관심 종목이 2개 이상 등록된 상태에서 "점수 비교" 화면을 열면, 각 종목의 단기·중기·장기 점수(0–100)와 점수 산출 근거 요약이 표시된다.

**Acceptance Scenarios**:

1. **Given** 사용자가 3개의 관심 종목(AAPL, TSLA, NVDA)을 등록한 상태에서, **When** "점수 비교" 화면을 열면, **Then** 각 종목의 단기·중기·장기 점수가 표 또는 카드 형태로 나란히 표시된다.
2. **Given** 비교 화면에서 특정 종목의 점수를 클릭했을 때, **When** 상세 정보가 펼쳐지면, **Then** 해당 점수의 산출 근거(어떤 지표가 긍정/부정으로 반영되었는지)가 텍스트로 표시된다.
3. **Given** 점수 비교 화면에서, **When** 어느 위치에서든, **Then** "점수는 AI 예측 참고용이며, 투자 결정의 책임은 사용자에게 있습니다"라는 면책 고지가 표시된다.
4. **Given** 관심 종목이 0개인 상태에서 비교 화면에 접근하면, **When** 화면이 로드되면, **Then** "관심 종목을 먼저 추가해 주세요" 안내 메시지와 종목 검색 링크가 표시된다.

---

### Edge Cases

- 시장 휴장일(주말, 공휴일)에 접속 시 "현재가"가 마지막 종가임을 명시해야 한다.
- 신규 상장 종목(역사적 데이터가 충분하지 않은 경우) — AI 분석 가능 여부 및 최소 데이터 요건을 명시해야 한다.
- 동일한 이름의 종목이 여러 거래소에 존재할 경우(예: 동명의 NASDAQ, KOSPI 종목) — 거래소 명을 함께 표시하여 사용자가 구분할 수 있어야 한다.
- 관심 종목 최대 등록 수 — 서비스 초기에는 적절한 상한선을 설정하여 과도한 API 호출을 방지해야 한다.
- AI 분석 요청이 동시에 대량 발생하는 경우 — 사용자에게 대기 시간을 안내하고 요청 상태를 표시해야 한다.
- 모바일 환경에서 차트 표시 — 화면 크기에 맞게 차트가 반응형으로 렌더링되어야 한다.

## Requirements *(mandatory)*

### Functional Requirements

**종목 검색 & 대시보드**

- **FR-001**: 시스템은 사용자가 종목명 또는 티커 심볼로 주식을 검색할 수 있어야 한다. 검색 결과는 미국 주식(NASDAQ, S&P500)과 한국 주식(KOSPI)을 모두 포함해야 한다.
- **FR-002**: 시스템은 검색된 종목의 현재가, 전일 대비 등락률(%), 거래량, 시가총액을 표시해야 한다.
- **FR-003**: 시스템은 1개월, 3개월, 6개월, 1년 단위의 주가 차트를 제공해야 하며, 사용자가 탭으로 기간을 전환할 수 있어야 한다.
- **FR-004**: 시장 데이터가 오래되었거나 불가능한 경우, 시스템은 마지막 수신 시각(타임스탬프)을 함께 표시하고 실시간 데이터가 아님을 사용자에게 명확히 알려야 한다.

**AI 분석 & 예측**

- **FR-005**: 시스템은 이동평균선(MA), RSI, MACD, 볼린저 밴드 지표 데이터를 산출하여 AI 분석의 입력으로 제공해야 한다.
- **FR-006**: AI 분석 리포트는 단기(1주) 전망과 중기(3개월) 전망을 별도 섹션으로 구분하여 텍스트로 제공해야 한다.
- **FR-007**: AI 분석 결과 화면에는 항상 "본 분석은 투자 참고용이며, 투자 판단의 책임은 사용자에게 있습니다"라는 면책 고지가 포함되어야 한다.
- **FR-008**: AI 분석에 필요한 최소한의 기술 지표 데이터가 확보되지 않은 경우, 시스템은 분석 대신 데이터 부족 안내 메시지를 표시해야 한다.

**관심 종목 관리**

- **FR-009**: 로그인한 사용자는 종목을 관심 종목으로 등록 및 해제할 수 있어야 한다.
- **FR-010**: 시스템은 관심 종목 요약 화면에서 등록된 모든 종목의 현재가, 등락률, 거래량을 한눈에 볼 수 있도록 표시해야 한다.
- **FR-011**: 비로그인 사용자가 관심 종목 기능에 접근하면 로그인 또는 회원가입 화면으로 안내해야 한다.

**AI 챗봇**

- **FR-012**: 시스템은 종목 관련 자연어 질문을 입력받아 AI가 해당 종목 데이터를 기반으로 자연어로 응답하는 챗봇 인터페이스를 제공해야 한다.
- **FR-013**: 챗봇은 주식 분석 범위를 벗어난 질문(예: 날씨, 일반 상식)에 대해 범위 밖임을 안내해야 한다.
- **FR-014**: 챗봇 응답 중 투자 판단에 영향을 줄 수 있는 내용에는 면책 고지가 포함되어야 한다.

**관심 종목 예측 점수 비교**

- **FR-015**: 시스템은 관심 종목으로 등록된 주식들에 대해 단기(1주), 중기(3개월), 장기(1년) 각각의 AI 예측 점수를 0–100점으로 산출해야 한다.
- **FR-016**: 점수 비교 화면에서 사용자는 모든 관심 종목의 단기·중기·장기 점수를 나란히 비교할 수 있어야 한다.
- **FR-017**: 각 점수를 클릭하면 해당 점수의 산출 근거(긍정/부정 요인 요약)를 확인할 수 있어야 한다.

**공통**

- **FR-018**: 모든 화면은 모바일 기기에서도 정상적으로 표시되는 반응형 웹 디자인을 적용해야 한다.
- **FR-019**: 사용자 계정 생성 및 로그인 기능을 제공해야 한다 (관심 종목, 챗봇 이력 저장에 필요).

### Key Entities *(include if feature involves data)*

- **Stock (종목)**: 거래소(NASDAQ/S&P500/KOSPI), 티커 심볼, 종목명, 현재가, 등락률, 거래량, 시가총액, 주가 이력 데이터
- **TechnicalIndicators (기술 지표)**: 종목 ID, 기준 일자, 이동평균선(5/20/60일), RSI(14일), MACD, 볼린저 밴드(상단/중단/하단), SMA(5/20/50/200일) + 스토캐스틱
- **User (사용자)**: 계정 ID, 이메일, 관심 종목 목록
- **Watchlist (관심 종목)**: 사용자 ID, 종목 ID, 등록 일시
- **AIReport (AI 분석 리포트)**: 종목 ID, 생성 일시, 단기 전망 텍스트, 중기 전망 텍스트, 사용된 지표 데이터 스냅샷
- **PredictionScore (예측 점수)**: 종목 ID, 생성 일시, 단기 점수, 중기 점수, 장기 점수, 각 점수의 근거 요약 텍스트
- **ChatMessage (챗봇 메시지)**: 사용자 ID, 종목 ID(선택), 질문 텍스트, AI 응답 텍스트, 생성 일시

## Success Criteria *(mandatory)*

### Measurable Outcomes

- **SC-001**: 사용자가 종목을 검색한 후 1분 이내에 AI 분석 리포트를 받을 수 있다.
- **SC-002**: 관심 종목으로 등록된 모든 종목의 단기·중기·장기 예측 점수(0–100점)를 단일 화면에서 확인할 수 있다.
- **SC-003**: AI 분석 내용은 반드시 실제 기술 지표 데이터(RSI, MACD 등 수치)를 포함하여 분석 근거를 제시해야 하며, 데이터 없이 생성된 분석은 0%여야 한다.
- **SC-004**: 모바일 기기(화면 폭 375px 이상)에서 모든 핵심 화면(검색, 대시보드, AI 분석, 관심 종목, 챗봇, 비교 점수)이 가로 스크롤 없이 정상 표시된다.
- **SC-005**: 면책 고지는 AI 분석 리포트, 챗봇 응답, 예측 점수 화면의 100%에서 표시된다.
- **SC-006**: 사용자가 처음 방문하여 종목 검색 → 대시보드 확인 → AI 분석 요청의 핵심 흐름을 5분 이내에 완료할 수 있다.

## Assumptions

- 미국 주식(NASDAQ, S&P500)이 주요 대상이며, 한국 주식(KOSPI)은 부가 지원 범위다.
- 사용자는 기본적인 차트 해석은 가능하지만 전문 트레이더 수준의 금융 지식은 없는 개인 투자자다.
- 주가 데이터 및 기술 지표 계산을 위한 외부 시장 데이터 API 또는 데이터 소스를 이용 가능하다고 가정한다.
- AI 분석 기능은 Claude API를 통해 구현하며, 입력으로 기술 지표 수치 데이터를 제공한다.
- 사용자 계정 관리(회원가입, 로그인)는 NextAuth.js v5 OAUTH로 구현
(구글, 카카오 연동)
- 관심 종목의 최대 등록 수는 초기 버전(v1)에서 30개로 제한한다.
- 초기 버전에서 실시간 스트리밍 주가(websocket 방식의 tick-by-tick)는 지원하지 않으며, 주기적 갱신(polling) 방식을 사용한다.
- 모바일 앱(iOS/Android 네이티브)은 범위 밖이며, 반응형 모바일 웹으로 지원한다.
- AI 분석 리포트는 요청 시점에 즉시 생성되며, 캐싱 정책(예: 동일 종목 1분 이내 재요청 시 캐시 반환)은 구현 단계에서 결정한다.
