**Feature Branch**: `003-ui-redesign`

**Status**: Draft

## User Scenarios & Testing *(mandatory)*

### User Story 1 - 햄버거 드로어 배경 불투명화 (Priority: P1)

사용자가 모바일(Galaxy S25 Edge 등)에서 상단 햄버거를 눌러 네비게이션 드로어를 열면, 현재는 반투명 배경(`bg-slate-950/95`)을 통과해 뒤 페이지의 콘텐츠(카드, 표, 차트 등)가 비쳐 보여 메뉴 텍스트의 시인성이 떨어진다. 드로어 배경을 완전히 불투명한 다크 톤으로 바꿔 메뉴 링크 글자가 명확히 읽히도록 한다.

**Why this priority**: 현재 모바일 사용자가 메뉴를 읽기 어려워 네비게이션이 실질적으로 막혀 있다. 이는 US-005 모바일 반응형 작업의 직접적인 후속 이슈이며, 접근성·가독성 문제로서 P1 차단 요소이다.

**Independent Test**: Galaxy S25 Edge 또는 Chrome DevTools iPhone SE(375px) 뷰포트에서 `/dashboard` 진입 → 카드가 채워진 화면을 확인 → 햄버거 탭 → 드로어가 열렸을 때 뒤 카드 콘텐츠가 전혀 비쳐 보이지 않고 메뉴 링크 글자가 충분한 대비로 표시되는지 독립 검증.

**Acceptance Scenarios**:

1. **Given** 대시보드 카드가 가득 찬 모바일 화면에서, **When** 햄버거 버튼을 탭하여 드로어를 열면, **Then** 뒤 배경 콘텐츠는 전혀 보이지 않아야 하며 메뉴 링크 텍스트가 명확히 읽혀야 한다.
2. **Given** 햄버거 드로어가 열린 상태에서, **When** 배경 스크롤을 시도하면, **Then** 뒤 페이지 스크롤은 잠겨 있고 드로어 내부 메뉴만 상호작용 가능하다.
3. **Given** 드로어가 열린 상태에서, **When** 링크를 탭하면, **Then** 해당 페이지로 이동하고 드로어는 자동으로 닫힌다 (기존 동작 유지).

### User Story 2 - AI 종합 투자 점수 UI 중앙 정렬 (Priority: P1)

사용자가 종목 상세 페이지(`/stock/[ticker]`)에서 AI 분석을 요청한 뒤 단기/중기/장기 점수 게이지가 표시될 때, 현재는 3개 게이지(`ArcGauge`, 각 116px) + 2개 세로 구분선이 합계 ~350px의 너비를 요구해 Galaxy S25 Edge 폭(~375px 화면에서 패딩 적용 후 약 328–340px 내부 폭)에서 오른쪽으로 밀려 보이고 중앙에 오지 않는다. 모바일에서도 항상 화면 중앙에 3개 게이지가 정렬되어 보이도록 한다.

**Why this priority**: AI 점수는 이 앱의 핵심 결과물이며, 현재 사용자가 실제 기기(Galaxy S25 Edge)에서 비정상 정렬을 확인한 상태이다. 브랜드 인상·정보 전달 모두에 악영향을 주므로 P1.

**Independent Test**: Galaxy S25 Edge(또는 DevTools 360px/375px/412px 뷰포트)에서 AAPL 분석 완료 → 단기/중기/장기 게이지 3개가 화면 수평 중앙에 대칭으로 배치되는지 독립 검증. 데스크톱 ≥640px에서는 기존 레이아웃이 깨지지 않아야 한다.

**Acceptance Scenarios**:

1. **Given** 모바일 뷰포트(< 640px)에서 AI 분석이 완료된 상태, **When** 점수 게이지 섹션을 보면, **Then** 단기/중기/장기 게이지가 한 줄에 가로 스크롤 없이 배치되고 섹션이 화면 수평 중앙에 위치한다.
2. **Given** 모바일 뷰포트, **When** 점수 게이지 섹션을 보면, **Then** 각 게이지의 "단기 (1주)" / "중기 (3개월)" / "장기 (1년)" 라벨이 모두 한 줄로 읽히고 잘리지 않는다.
3. **Given** 데스크톱 뷰포트(≥ 640px), **When** 동일 화면을 보면, **Then** 기존 게이지 크기·간격·라벨이 회귀 없이 동일하게 표시된다.

### User Story 3 - /history 뱃지 한 줄 고정 (Priority: P2)

사용자가 `/history` 분석 기록 테이블을 볼 때, 유형(비교/종목) 및 단기/중기/장기 뱃지(중립·매수고려·매수·매도 등 한글 라벨)가 모바일 좁은 화면에서 셀 너비를 넘어가면 2줄로 줄바꿈되어 테이블이 지저분해 보인다. 뱃지 텍스트는 무조건 한 줄로 표시되어야 한다.

**Why this priority**: 가독성 저하이지만 테이블 기능 자체는 동작한다. 햄버거·점수 UI 다음 순위의 다듬기 작업이므로 P2.

**Independent Test**: `/history` 페이지에 여러 유형·점수의 기록이 있는 상태에서 모바일 뷰포트로 진입 → 모든 "비교"/"종목"/"중립"/"매수고려"/"매수"/"매도" 뱃지가 줄바꿈 없이 한 줄로 표시되는지 독립 검증. 뱃지 라벨이 셀 폭을 넘는 경우 가로 스크롤(테이블 래퍼에 이미 존재)로 처리된다.

**Acceptance Scenarios**:

1. **Given** `/history`에 분석 기록이 있는 상태, **When** 모바일에서 페이지를 열면, **Then** 유형·단기·중기·장기의 모든 뱃지 텍스트가 줄바꿈 없이 한 줄에 표시된다.
2. **Given** 뱃지 폭 합계가 테이블 min-width를 초과할 때, **When** 테이블을 가로로 스크롤하면, **Then** 뱃지는 여전히 한 줄을 유지하며 가로로 노출된다.

### User Story 4 - /scores 토글 4개가 모바일 한 화면에 배치 (Priority: P2)

사용자가 `/scores` 점수 랭킹 페이지에서 상단 정렬 토글(단기 (1주) / 중기 (3개월) / 장기 (1년) / 종합)을 볼 때, 현재는 4개 버튼이 가로로 나열되면서 모바일 너비를 초과해 가로 스크롤이 발생한다. 모바일에서도 가로 스크롤 없이 4개 토글이 한 화면에 보이도록 폰트 크기와 패딩을 조정한다.

**Why this priority**: 사용자가 토글을 인지하지 못하면 랭킹 정렬 기능을 발견하지 못한다. 기능 동작에는 문제가 없지만 발견성 저하이므로 P2.

**Independent Test**: Galaxy S25 Edge 뷰포트에서 `/scores` 진입 → 토글 4개가 모두 같은 행에 보이고 가로 스크롤이 발생하지 않는지, 각 토글이 클릭 가능한 면적을 유지하는지(터치 타겟 ≥ 36px 세로) 독립 검증.

**Acceptance Scenarios**:

1. **Given** 모바일 뷰포트(< 640px), **When** `/scores`를 열면, **Then** 4개의 정렬 토글이 가로 스크롤 없이 한 행에 모두 보인다.
2. **Given** 모바일에서 토글을 보면, **When** 각 토글을 탭하면, **Then** 정렬 결과가 정상 반영되고 선택된 토글이 시각적으로 활성화 표시된다.
3. **Given** 데스크톱 뷰포트(≥ 640px), **When** 동일 페이지를 보면, **Then** 기존 토글 라벨(괄호 포함 "단기 (1주)" 등)과 레이아웃이 동일하게 유지된다.

### User Story 5 - 차트 그린 선 hover/터치 가격 표시 (Priority: P3)

사용자가 종목 차트에서 추세선 또는 수평선을 그린 뒤, 그 선에 마우스를 올리거나(데스크톱) 모바일에서 터치하면, 현재 커서 위치에 해당하는 가격을 툴팁으로 표시한다. 수평선은 이미 축 라벨이 있지만 추세선은 좌표별 가격이 없으므로 사용자가 "이 지점에서 몇 달러?"를 확인할 수 없다.

**Why this priority**: 차트 그리기 기능이 이미 동작하고 있고, 가격 표시는 정교화 단계의 개선이므로 P3. 하지만 구현되면 기능의 실용성이 크게 향상된다.

**Independent Test**: 차트에 추세선을 하나 그린 뒤, 그 선 근처에 마우스를 올리거나(데스크톱) 터치(모바일)하여 "해당 x(시간) 좌표에서 그 선이 가지는 y(가격)" 값이 툴팁으로 표시되는지 독립 검증. 기존 크로스헤어 툴팁(봉의 OHLC)과 동시 표시 시 충돌 없이 병존해야 한다.

**Acceptance Scenarios**:

1. **Given** 추세선이 그려진 차트, **When** 그 선 위에 마우스를 올리면, **Then** 커서 위치의 x좌표에서 선이 가지는 가격이 툴팁으로 표시된다.
2. **Given** 추세선이 그려진 차트를 모바일에서 볼 때, **When** 선 위를 터치하면, **Then** 터치 지점의 x좌표에서 선이 가지는 가격이 툴팁으로 표시된다.
3. **Given** 수평선이 그려진 차트, **When** 선 위에 마우스 올리거나 터치하면, **Then** 해당 고정 가격이 표시된다 (축 라벨과 중복돼도 무방).
4. **Given** 드로잉 모드가 OFF인 상태, **When** 차트 일반 영역을 호버하면, **Then** 기존 봉 크로스헤어·툴팁이 회귀 없이 동작한다.

---

### Edge Cases

- **햄버거 드로어**: 드로어 열린 상태에서 화면 회전(세로↔가로) 시 배경은 계속 불투명 유지. 닫기 애니메이션 중간 프레임에서도 배경 비침 없음.
- **점수 게이지 중앙 정렬**: 아주 좁은 뷰포트(예: 320px)에서는 3개 게이지가 세로로 스택되더라도 중앙 정렬 유지. 태블릿(640px 근처 경계)에서 깨지지 않아야 함.
- **/history 뱃지**: 점수 라벨이 이례적으로 긴 경우(예: "적극매수고려" 같은 확장) 셀 너비가 늘어날 수 있음 → 테이블 래퍼의 가로 스크롤이 흡수.
- **/scores 토글**: 토글 폰트를 줄여도 접근성 기준(최소 12px 이상) 준수. 터치 타겟 최소 세로 36px 유지.
- **차트 선 호버 툴팁**: 여러 선이 겹치는 지점에서는 가장 가까운 선 하나의 가격만 표시(목록 X). 터치 이벤트는 단일 탭(탭-앤-홀드 X)으로 가격 표시, 이후 다른 곳 탭 시 숨김.

## Requirements *(mandatory)*

### Functional Requirements

- **FR-001**: 햄버거 드로어는 열렸을 때 뒤 페이지가 비치지 않는 완전 불투명 배경(Tailwind: `bg-slate-950`)을 사용하며, 상단에 약한 그라디언트 엣지(`before:bg-gradient-to-b from-purple-500/10 to-transparent` 또는 동등한 시각 효과)를 허용하여 브랜드 톤을 유지한다.
- **FR-002**: 햄버거 드로어는 열렸을 때 배경 body 스크롤을 잠그고(`overflow-hidden`), 링크 탭 시 자동 닫힘, `Escape` 키 또는 상단 닫기 버튼으로 수동 닫힘을 지원한다.
- **FR-003**: `/history` 페이지의 모든 뱃지 컴포넌트(`TypeBadge`, `ScoreBadge`)는 `whitespace-nowrap`을 가지며 어떤 너비에서도 줄바꿈되지 않는다.
- **FR-004**: `/scores` 정렬 토글은 모바일(< 640px)에서 4개가 한 행에 배치되어야 한다 (가로 스크롤 금지). 모바일에서는 축약 라벨 "단기 / 중기 / 장기 / 종합"을, 데스크톱(≥ 640px)에서는 기존 라벨 "단기 (1주) / 중기 (3개월) / 장기 (1년) / 종합"을 사용한다.
- **FR-005**: `/scores` 토글의 모바일 폰트 크기는 최소 12px(`text-xs`) 이상이며 터치 타겟 세로 높이는 최소 36px(`py-2` 이상 + 라인하이트) 이상을 유지한다.
- **FR-006**: 종목 상세 페이지(`/stock/[ticker]`)의 AI 점수 게이지 섹션은 모바일(< 640px)에서 화면 수평 중앙에 정렬되며, 3개 게이지가 가로 스크롤 없이 한 행에 배치된다. 필요 시 게이지 크기(`w-28 h-28` → `w-24 h-24`)를 모바일에서 축소한다.
- **FR-007**: 종목 상세 AI 점수 섹션은 데스크톱(≥ 640px)에서 기존 레이아웃·크기·라벨을 회귀 없이 유지한다.
- **FR-008**: 차트에 그려진 추세선에 데스크톱 마우스 호버 또는 모바일 터치 이벤트가 발생하면, 해당 x(시간) 좌표에서 선이 가지는 y(가격)을 계산해 툴팁으로 표시한다. 수평선은 자신의 고정 가격을 표시한다.
- **FR-009**: 차트 선 호버 툴팁은 기존 봉 크로스헤어 툴팁과 공존하며, 드로잉 모드가 OFF일 때는 기존 봉 호버 동작을 회귀 없이 유지한다.
- **FR-010**: 모든 변경은 Constitution §II Disclaimer, §Data&Privacy 요구사항에 영향을 주지 않으며 백엔드 API 스펙을 변경하지 않는다 (프론트엔드 전용 리팩토링).

### Key Entities

- **Drawer State (client)**: 햄버거 드로어 열림 여부, 활성 라우트. 기존 `NavClient.tsx`의 `mobileOpen` 상태 확장 없음 — 배경 클래스만 교체.
- **ChartDrawing (client-persisted)**: 기존 `{ticker}_{market}` 키 하 localStorage 구조 그대로 유지. 호버 툴팁은 읽기 전용으로 해당 데이터를 사용한다.
- **ScoreToggle (client)**: `/scores` 정렬 선택. 기존 상태 그대로 유지하고 렌더링 분기만 `sm:` 브레이크포인트로 변경.

## Success Criteria *(mandatory)*

### Measurable Outcomes

- **SC-001**: Galaxy S25 Edge(또는 Chrome DevTools 360–412px 뷰포트)에서 햄버거 드로어를 열었을 때 뒤 콘텐츠 픽셀 휘도 변화율 0%(완전 차단).
- **SC-002**: `/history` 페이지의 모든 뱃지가 뷰포트 320px–1920px 범위에서 0건의 줄바꿈을 보인다.
- **SC-003**: `/scores` 페이지가 360px 뷰포트에서 토글 4개가 가로 스크롤바(`::-webkit-scrollbar` 표시 또는 `scrollWidth > clientWidth`) 없이 렌더링된다.
- **SC-004**: 종목 상세 페이지 AI 점수 섹션이 360px–412px 뷰포트에서 좌우 여백 차이 8px 이내의 대칭 중앙 정렬을 유지한다.
- **SC-005**: 추세선 위 호버(또는 터치) 시 툴팁이 100ms 이내에 표시되며 좌표 기반 가격 오차가 실제 픽셀→가격 변환 정밀도 범위 내이다.
- **SC-006**: 데스크톱(≥ 1280px) 회귀: `/dashboard`, `/history`, `/scores`, `/stock/[ticker]`의 기존 레이아웃·라벨·차트 기능이 시각적으로 변경되지 않는다.

## Assumptions

- 대상 주요 디바이스는 Galaxy S25 Edge(≈ 375px 내부 폭) 및 iPhone SE(375px)를 포함한 360–414px 모바일 구간이다.
- 브라우저는 최신 Chrome/Edge/Safari를 가정하며 CSS `backdrop-blur`, Tailwind `sm:` 브레이크포인트, `pointer-events` 제어가 모두 지원된다.
- 002-ui-redesign의 햄버거 NAV, 차트 드로잉, /history 행 클릭, AI 점수 배지, 토큰 로깅 변경이 이미 머지되어 동작 중이다.
- 백엔드 API와 DB 스키마는 변경하지 않으며, 본 피처는 프론트엔드 UI 튜닝만을 대상으로 한다.
- 기존 Tailwind 유틸리티 클래스 기반 스타일링 규칙을 유지하며, 새 CSS 파일은 추가하지 않는다.
